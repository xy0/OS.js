Development Introduction

There's some manual pages that explains some aspects of the development process of OS.js:

  • Configuration system
  • Build system

Below you can find some information on how to get started with developing for OS.js:

Setting up a development environment

OS.js already comes with a development environment, using dist-dev.

To install all utilities, do: 

# Install all grunt tasks etc
$ npm install --development

# To automatically restart server when changes in codebase occur
$ npm install -g supervisor

# Unit testing support
$ sudo npm install -g mocha

Running the development server

Just use the launcher: 

./bin/start-dev.sh

How to contribute

If you want to contribute to the official codebase, this is a basic overview of all the steps:

  • Create a Github account.
  • Fork OS.js to your account
  • Proceed to clone your repository on your computer
  • Create a new branch (ex: 'translate-turkish')
  • Make any changes you want. Follow the style guide and test everything (see below for more info).
  • Commit any changes you have and push them.
  • Create a pull-request on Github.

Make sure that your commit messages are useful and to the point.. You can look at the OS.js repository for examples.

If you make multiple commits regarding one feature, or bug, make sure to squash them.

If you don't follow these rules or steps, your pull-request might be rejected

You can switch back to master branch (and pull) after your changes has been merged. If you want to make more changes, just re-follow the steps above (after forking)

Style guide

  • Use UTF-8 for your file encoding
  • Use 2 spaces, not tabs for indentation.

You can check your code style for "problems" by using automated tools: 

# Check your javascript
grunt jshint jscs

# Check your css
grunt csslint

# Check your schemes (html)
grunt xml_validator

# Run server unit tests
grunt mochaTest

Tips

Restart your application

You can reload and relaunch your application with: 

OSjs.API.relaunch('MyApplicationClassName')

Automatically restart server on changes

Run this grunt task to automatically restart server when codebase or configs changes (with exception of apps): 

grunt watch

TEST ALL THE THINGS!

To run all tests on the code, just use: 

grunt test
