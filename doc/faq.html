If you want to submit your own questions just go to the Community Page, Gitter Chatroom, or social media.

I strongly recommend browsing the manuals and tutorials if you're new to the OS.js environment.

Q: Why was OS.js created?

OS.js was originally intended for configuring (web and storage) servers, but turned into a desktop platform over the years.

People ask why? And isn't this a waste of time?... and I really have no answer except for:

Because I can.. and nothing is a waste of your time as long as you learn something in the process. And I've learned a lot! This is something I like to do in my spare-time because it's damn fun.

Q: What are the use cases for OS.js?

With recent web technologies, anything is possible. With OS.js you can run multiple web applications within an environment that most users are familar with (the "desktop experience").

  • Useful to communicate with underlying systems with GUIs
  • Modular backoffice-, CMS- or CRM solutions
  • Intranet solutions
  • Remote Desktop (and shell) over web
  • Runs on underpowered hardware and pretty much any platform.
  • Single-board-computer solutions, as well as IoT projects
  • Cloud development and testing platforms
  • Monitoring of systems (think graphs and sexy data)

Q: Is it production ready?

The current state is an Alpha, but that does not mean it's not ready for real-world usage. Some things in the APIs might change, but backwards-compability is kept for a while so you have time to update your code.

If you're wondering about security, this is where I need your help. If you're familiar with web-security and know how to audit and/or how to improve the code, please let me know. I usually get anonymous emails when a security issue has been discovered, and they are patched within 24 hours. If you find one, let me know at andersevenrud@gmail.com, or make an issue on Github.

Any news and notices regarding security will be posted on the Community Website.

Q: How do I brand my distribution?

The build system supports templates for branding and other customizations.

Simply copy the base template and modify it as you want. 

cp src/templates/dist/default src/templates/dist/mybrand
grunt config --set=dist.template --value=mybrand
grunt

Q: Why do I get these errors when installing node dependencies?

First, make sure you're running the latest node and npm version available for your system.

Windows users: Make sure you have npm 3.8.3 or later! Or else you will run into problems with exceeding the maximum filename lengths of the NTFS filesystem. 

# Remove the cache
npm cache clean

# Remove installed modules
rm -rf node_modules

# Reinstall modules (if you're a developer, skip the production flag)
npm install --production
